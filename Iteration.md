# 游戏秘籍快捷输入工具 - 迭代计划文档

## 项目概述
本文档定义了 GameCheatHelper 项目的迭代开发计划，采用敏捷开发方法，分为多个迭代周期进行增量开发。

---

## 迭代策略

### 开发模式
- **迭代周期**: 1周/迭代
- **开发方法**: 敏捷开发 (Scrum-inspired)
- **版本策略**: 语义化版本 (SemVer)

### 版本号规则
- **主版本 (Major)**: 重大功能更新或架构变更
- **次版本 (Minor)**: 新功能添加
- **修订版本 (Patch)**: Bug 修复和小改进

---

## 迭代 1: MVP 核心功能 (v0.1.0)

### 迭代目标
构建最小可用产品 (MVP)，实现基础的秘籍输入功能。

### 时间范围
第 1 周

### 功能范围

#### 必须完成 (Must Have)
- [x] 项目初始化和架构搭建
- [x] Windows API 封装 (基础功能)
- [x] 键盘输入模拟器
- [x] 游戏进程检测 (魔兽争霸3)
- [x] 热键管理器 (基础版)
- [x] 秘籍执行引擎
- [x] 简单的 UI 界面 (WPF)

#### 应该完成 (Should Have)
- [x] 基础日志系统
- [x] 基础异常处理

#### 可以完成 (Could Have)
- [ ] UI 美化
- [ ] 配置文件支持

#### 不会完成 (Won't Have)
- 高级功能 (自定义秘籍、云同步等)

### 交付物
- 可运行的 WPF 应用程序
- 支持检测魔兽争霸3
- 支持 3-5 个预设秘籍
- 基础热键绑定 (F1-F5)

### 验收标准
- [ ] 能够检测到运行中的魔兽争霸3
- [ ] 按下 F1 能够成功输入 "greedisgood"
- [ ] 应用程序无崩溃运行
- [ ] 有基本的错误提示

### 技术债务
- UI 界面较简陋
- 无配置持久化
- 热键硬编码

### 风险
- **风险**: Windows API 调用失败
  - **缓解**: 充分测试，添加错误处理
